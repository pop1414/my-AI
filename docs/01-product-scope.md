# 产品范围说明（Product Scope）

## 1. 项目目标
构建一个基于 Spring AI 的智能文档问答系统，采用分阶段演进方式：先跑通核心链路，再逐步增强多模型、多知识库、多租户能力。

## 2. 目标用户
- 个人开发者（当前阶段）
- 小团队内部知识问答（后续）
- 企业租户（SaaS 阶段）

## 3. 当前版本范围（V1）
### In Scope（本版本要做）
- 单用户模式
- 支持 TXT/PDF 文档上传与解析
- 文档分块、向量化、向量入库
- 基于 RAG 的问答接口
- 固定使用一个 LLM Provider（OpenAI）
- 固定使用一个向量存储（PostgreSQL + PGVector）

### Out of Scope（本版本不做）
- 多租户隔离
- 多模型供应商动态切换
- 企业自定义 API Key
- 计费与配额
- 复杂权限系统

## 4. 成功标准（DoD）
- 文档上传成功率 >= 95%
- 能针对已上传文档进行可复现问答
- 本地一条命令可启动（含依赖说明）
- 关键链路有基础日志与错误处理

## 5. 约束与假设
- 以“完成比完美更重要”为原则
- 先保证端到端可用，再做优化
- 架构上预留 Provider / Store 扩展接口，但 V1 只实现一个具体实现

## 6. 风险清单（初版）
- PDF 解析质量不稳定
- 向量检索召回质量不足
- 外部模型 API 波动导致响应不稳定

## 7. 里程碑验收
- M1：文档入库链路打通（上传 -> 分块 -> 向量入库）
- M2：问答链路打通（检索 -> 生成）
- M3：完成 V1 发布与版本归档
