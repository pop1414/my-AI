# ADR-0001：V1 技术基线

- 编号：ADR-0001
- 标题：V1 阶段固定单模型与单向量存储，优先打通端到端链路
- 状态：Accepted
- 日期：2026-03-30

## 背景
项目目前为单人开发，目标是在最短时间内完成可运行版本（V1），而不是一次性实现多模型、多租户、计费等完整能力。

## 备选方案
1. 一次性实现 V1/V2/V3 主要能力
2. 仅实现 V1 能力，并在架构层预留扩展接口

## 决策
选择方案 2：
- V1 固定 OpenAI + 固定向量库
- 业务层依赖抽象接口，保留后续扩展可能

## 决策理由
- 降低复杂度与交付风险
- 更快获得可验证反馈
- 通过抽象边界避免未来大规模重构

## 影响
### 正向影响
- 交付速度更快
- 调试与排错链路更短

### 负向影响 / 风险
- V1 灵活性有限
- V2 引入多 Provider 时仍有一定改造成本

## 后续动作
- 在 V1 完成后新增 ADR：Provider 插件化与会话记忆策略
- 新增 ADR-0002：明确 V1 具体模型与向量存储落地选择
