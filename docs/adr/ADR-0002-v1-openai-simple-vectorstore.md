# ADR-0002：V1 具体落地基线（OpenAI + 本地向量库）

- 编号：ADR-0002
- 标题：V1 使用 OpenAI 作为唯一模型提供方，使用本地向量库存储索引
- 状态：Accepted
- 日期：2026-03-31

## 背景
`ADR-0001` 已确定 V1 采用“单模型 + 单向量存储”策略，但尚未明确具体实现。  
当前目标是优先跑通 M1（上传 -> 分块 -> 向量入库），并降低环境搭建复杂度，便于学习和调试。

## 备选方案
1. OpenAI + Redis Vector（需要额外中间件）
2. OpenAI + 本地向量库（SimpleVectorStore，进程内/本地文件）
3. 直接引入多 Provider + 多 VectorStore

## 决策
选择方案 2：
- LLM/Embedding：固定 OpenAI
- Vector Store：固定本地向量库（SimpleVectorStore）

## 决策理由
- 本地运行门槛最低，适合当前学习阶段
- 减少外部依赖，能更快定位链路问题
- 与 V1 “先跑通端到端，再扩展”的目标一致

## 影响
### 正向影响
- 无需先部署 Redis/Milvus，即可验证 RAG 基础链路
- 开发与调试反馈更快

### 负向影响 / 风险
- 本地向量库不适合生产场景
- 数据持久化与并发能力有限，后续仍需迁移到专业向量库

## 后续动作
- M1 阶段基于该基线完成上传与入库链路
- 进入 V2 前补充迁移 ADR（例如迁移至 Redis/PGVector）
